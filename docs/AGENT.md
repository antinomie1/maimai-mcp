# maimai-mcp Agent 规则（通用）

任意 LLM / Agent 宿主调用本仓库 MCP 工具时，请遵循本文。可整段写入系统提示或工具说明。

## 对用户回复的隐私硬规则（最高优先级）

**面向用户的最终回复中，禁止出现：**

- 任何 **QQ 号**（纯数字长串、`QQ=…`、昵称后的 `/数字`）
- 任何 **群号**、`group_id`、会话 ID 中的数字身份
- 工具原始 JSON 中的 `default_qq` / `group_id` / `qq` 字段原文
- 含 QQ 的本地路径念读（如 `b50_…….png`）——只说「已生成图片」或直接发图，不强调文件名中的数字

**允许：** 曲名、难度、分数、rating、FC/AP、牌子/版本名、称呼（「你」「该玩家」「对方」）、图片本身。

身份数字 **只用于工具参数**，不要复述给用户。

错误时用人话，例如：「查不到该玩家成绩，可能未绑定查分器。」——**不要**把错误的 QQ 数字念出来。

## 目标解析（工具侧）

| 用户说法 | 工具参数 `qq` |
|----------|----------------|
| 我的 / 帮我查 / 未指明对象 | **发送者** 玩家 ID（宿主事件中的用户标识，**不是群号**） |
| @ 某人 / 他的 / 对方的 | **被 @ 用户** 的玩家 ID |
| 明确水鱼用户名 | 可传 `username`（缺玩家 ID 时） |

**群号绝不写入 `qq`。** 会话形态常见为 `…:用户ID_群号` 或前缀 `昵称/用户ID`：用户 ID 与群号分开，后者仅可作 `group_id`。

MCP **不会**自动注入聊天上下文；能确定玩家时，查分类工具必须传 `qq`，或先调用 `maimai_set_identity`。

## 会话开场

同一次 MCP 进程内（若尚未设置）：

```text
maimai_set_identity
  qq = <发送者或目标玩家ID>    # 仅工具参数，勿对用户展示
  group_id = <当前群号>        # 可选；仅上下文，不当查分 qq
```

之后同进程可省略 `qq`（session 粘性）。**MCP 进程重启后需重新 set。**

若工具报「像是群号」：改为正确的玩家 ID，**不要**对错误 id 调用 `maimai_user_set_source` / `maimai_user_bind_lxns`。

## 常用工具

| 需求 | 工具 |
|------|------|
| 会话身份 | `maimai_set_identity` / `maimai_get_session` |
| B50 | `maimai_b50` |
| 等级/定数分数列表 | `maimai_score_list` |
| 牌子完成/进度 | `maimai_plate` |
| 定数表 | `maimai_rating_table` |
| 等级进度 | `maimai_level_progress` |
| 单曲成绩 | `maimai_minfo` |
| 谱面信息 | `maimai_chart` / `maimai_lookup_song` |
| 上分 | `maimai_rise` / `maimai_push_plan` |
| 曲库刷新 | `maimai_update_catalog` |

出图优先 `format: image`；将返回的图片交给宿主发送，**回复文案不写 QQ/群号**。

## 回复风格

- 好：「你的 13 分列表好了～」+ 图片  
- 好：「檄极进度如图。」+ 图片  
- 坏：「已为 QQ … 在群 … 查询」  
- 坏：把 `{"default_qq":…}` 贴给用户  

## 故障话术（对用户）

| 情况 | 说法 |
|------|------|
| 用户不存在 / 未绑定 | 查不到成绩，请确认查分器绑定与隐私设置 |
| 落雪未授权 | 需要先完成落雪授权绑定 |
| 曲库缺谱面 | 可提示更新曲库；勿向用户念内部 key |

排错时身份数字仅可出现在 **开发者日志**；**用户可见消息仍禁止**。
