# maimai-mcp 调用

调用任意 `maimai_*` 前先遵循本 skill。把机器人 QQ 当玩家或出图后不发图，是最常见错误。

## 何时调用

**仅当**用户要查舞萌成绩、歌曲、进度、曲库或运势时，才调用 `maimai_*`。

| 用户意图 | 是否调用 |
|----------|----------|
| b50 / 查分 / 完成表 / 牌子 / 分列表 / 定数表 / 上分 / 搜歌 / 谱面 / 运势 / 今日舞萌 | 是 |
| stop、闲聊、编码、修 bot、发消息超时 | 否 |
| 「为什么不发图」且上一轮已有 `image_path` | 否（直接补发该图片，不要重查） |

禁止：每条消息改设定；用查分「自救」发消息失败；校验失败后对调 `qq` 与群号。

## 调用格式（必读）

MCP **不会**读聊天上下文。参数必须包在 **`params` 对象**里；**一次只调一个** `maimai_*`。

硬性禁止：

- 空参数 `{}` 或漏掉 `params`
- 并行多个 `maimai_*`
- 用机器人自身 QQ 当 `params.qq`
- 出图成功后只回文字、不发图

可抄模板：

```text
工具: maimai_b50
参数:
  params:
    qq: <发送者 user_id>
    format: image
```

```text
工具: maimai_fortune
参数:
  params:
    qq: <发送者 user_id>
    format: image
```

## 查谁（唯一路径）

群聊里 **@机器人只是唤醒**，不是查分对象。

| 用户说法 | `params.qq` |
|----------|-------------|
| 我的 / 帮我查 / 查我 / 未指明对象 / 只 @ 了机器人 | **发送者** `user_id` |
| 明确 @ **另一个玩家** 查他的 | 被 @ 的**玩家** id |
| 只给水鱼用户名 | 用 `username`，可不写 qq |

```text
默认 = 发送者 user_id
@机器人 / 会话里的 bot 号 = 忽略，不当 qq
群号 = 绝不进 qq
```

从消息详情读取时：

- `发送者: … (3781884259)` → 这才是默认 `qq`
- 内容里 `@1938546323` 若是本 bot → **不要**当 `qq`
- 只有 @ 的是**别人**且用户要查「他的」时，才用被 @ 的 id

错误示例（日志里真实踩过）：用户 `@机器人 b50` → 模型传 `qq=机器人号` → 查分失败。  
正确：同一句应传 `qq=发送者号`。

主题 / 默认查分器：带对玩家 `qq` 后由库内自动读；**仅用户明确要求**时再 `maimai_user_set_*`。

## 出图必须发出去

查分/运势/表类默认 `format: image`。

工具成功且返回 **`image_path`** 时：

1. **同一轮回复必须**用宿主发图能力发出该路径（如 `send_message_to_user`，`type: image` + `path`）
2. 文案可简短（「好了～」），**不要**只贴路径或 JSON
3. **不要**把 QQ / 本地路径念给用户
4. 用户问「为什么不发图」：补发上一张 `image_path`，**不要**再调一次 `maimai_*`（除非路径已失效）

没有 `image_path` 且 `ok: false`：用人话说明原因，勿瞎换 qq 重试。

## 工具选型

完整表见 [references/tools.md](references/tools.md)。

| 需求 | 工具 |
|------|------|
| B50 | `maimai_b50` |
| 运势 / 今日舞萌 | `maimai_fortune` |
| 等级/定数分列表 | `maimai_score_list` |
| 牌子 | `maimai_plate` / `maimai_plate_status` |

| 上分 + 首选谱面 | `maimai_push_plan` |
| 定数表 / 等级进度 | `maimai_rating_table` / `maimai_level_progress` |

## 数据源与限流

默认水鱼。落雪易 429：让用户稍后再试；**仅用户明确要求**时切换数据源。

## 对用户回复

**不要出现：** QQ、群号、工具 JSON、含 QQ 的路径。  
**可以：** 曲名、难度、分数、rating、FC/AP、牌子名、「你」、**图片本身**。

| 情况 | 对人话 |
|------|--------|
| 未绑定 / 查无 | 查不到成绩，请确认查分器绑定与隐私设置 |
| 落雪未授权 | 需要先完成落雪授权绑定 |
| 请求过多 | 落雪查分过于频繁，请稍后再试 |
| 把机器人号当玩家 | （应避免；若发生）请用本人账号重试 |

## 附录【高级·默认跳过】

昵称反查：`maimai_resolve_qq`（需 `maimai_refresh_identity` + NapCat）。有发送者 `user_id` 时直接传，不要先反查。
